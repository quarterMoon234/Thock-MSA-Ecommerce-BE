Thock Kubernetes 배포 주의사항

1) 핵심 원칙
- 운영 배포에서는 image:latest 사용 금지
- 배포 태그는 merge commit SHA(또는 릴리스 태그)로 고정
- Kubernetes 수동 배포 시에는 반드시 `kubectl set image`로 서비스별 태그를 명시
- `services/*.yaml`의 `${IMAGE_TAG}`는 placeholder이므로 raw `kubectl apply -f services/...`만으로 이미지 배포가 완료되지 않음

2) 수동 배포 표준 절차
1. 매니페스트 반영 (초기 생성 또는 spec 변경 시에만)
   kubectl apply -f services/member-service.yaml
   kubectl apply -f services/product-service.yaml
   kubectl apply -f services/payment-service.yaml
   kubectl apply -f services/settlement-service.yaml
   kubectl apply -f services/market-service.yaml
   kubectl apply -f services/api-gateway.yaml

2. 이미지 태그 지정 (필수)
   NS=thock-prod

   # 예: 변경된 서비스만 배포
   kubectl -n $NS set image deployment/member-service \
     member-service=sang234/member-service:<member-service_SHA>

   # 예: 공용 모듈 변경으로 여러 서비스를 함께 배포
   kubectl -n $NS set image deployment/api-gateway \
     api-gateway=sang234/api-gateway:<api-gateway_SHA>
   kubectl -n $NS set image deployment/member-service \
     member-service=sang234/member-service:<member-service_SHA>
   kubectl -n $NS set image deployment/product-service \
     product-service=sang234/product-service:<product-service_SHA>

3. 롤아웃 확인
   kubectl -n $NS rollout status deployment/member-service --timeout=300s
   kubectl -n $NS wait --for=condition=ready pod -l app=member-service --timeout=180s

3) 배포 후 검증 (필수)
- 실제 실행 이미지 확인 (latest 금지 확인)
  kubectl -n thock-prod get deploy api-gateway member-service product-service market-service payment-service settlement-service \
    -o=jsonpath='{range .items[*]}{.metadata.name}{" => "}{.spec.template.spec.containers[0].image}{"\n"}{end}'

- Pod 상태 확인
  kubectl -n thock-prod get pods

- 헬스체크 확인
  kubectl -n thock-prod exec deploy/api-gateway -- sh -lc 'wget -qO- http://member-service:8081/actuator/health'
  kubectl -n thock-prod exec deploy/api-gateway -- sh -lc 'wget -qO- http://product-service:8082/actuator/health'
  kubectl -n thock-prod exec deploy/api-gateway -- sh -lc 'wget -qO- http://market-service:8083/actuator/health'
  kubectl -n thock-prod exec deploy/api-gateway -- sh -lc 'wget -qO- http://payment-service:8084/actuator/health'
  kubectl -n thock-prod exec deploy/api-gateway -- sh -lc 'wget -qO- http://settlement-service:8085/actuator/health'

4) 롤백 절차
- 직전 리비전 롤백
  kubectl -n thock-prod rollout undo deployment/member-service

- 또는 이전 정상 태그로 재지정
  kubectl -n thock-prod set image deployment/member-service \
    member-service=sang234/member-service:<이전_정상_SHA>

5) 운영 시 자주 발생하는 실수
- kubectl apply만 하고 set image를 하지 않음
- 공통 `IMAGE_TAG` 하나로 모든 서비스를 같은 릴리스라고 가정함
- latest 태그를 다시 사용함
- rollout status 확인 없이 배포 완료로 판단함
- 검증 없이 PR/배포 이슈를 종료함
