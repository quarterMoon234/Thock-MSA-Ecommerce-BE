Thock Docker Compose 수동 배포 주의사항

1) 핵심 원칙
- 운영 배포에서는 image:latest 사용 금지
- 배포 태그는 merge commit SHA(또는 릴리스 태그)로 고정
- 고정 설정은 `.env`, 서비스별 이미지 태그는 `release-state.env`에서 관리
- Compose 실행은 반드시 `./compose.sh`를 사용

2) 수동 배포 표준 절차
1. 배포 디렉토리 이동
   cd ~/deploy-docker-compose

2. 현재 배포 상태 백업
   cp release-state.env release-state.env.bak.$(date +%Y%m%d-%H%M%S)

3. 태그 지정 (release-state.env)
   예: MEMBER_SERVICE_IMAGE_TAG=<배포할_SHA>
   (변경 서비스만 수정, 나머지 서비스는 기존 값 유지)

4. 이미지 pull
   ./compose.sh pull <대상서비스>

5. 컨테이너 재기동
   ./compose.sh up -d <대상서비스>

6. 상태 확인
   ./compose.sh ps

3) 배포 후 검증 (필수)
- 실제 실행 이미지 확인 (latest 금지 확인)
  ./compose.sh config | grep "image:"
  docker ps --format "table {{.Names}}\t{{.Image}}"

- 헬스체크 확인
  docker exec api-gateway sh -lc 'wget -qO- http://member-service:8081/actuator/health'
  docker exec api-gateway sh -lc 'wget -qO- http://product-service:8082/actuator/health'
  docker exec api-gateway sh -lc 'wget -qO- http://market-service:8083/actuator/health'
  docker exec api-gateway sh -lc 'wget -qO- http://payment-service:8084/actuator/health'
  docker exec api-gateway sh -lc 'wget -qO- http://settlement-service:8085/actuator/health'

4) 롤백 절차
- 이전 정상 태그로 release-state.env 값을 되돌린 뒤 재배포
  예: MEMBER_SERVICE_IMAGE_TAG=<이전_정상_SHA>
  ./compose.sh pull <대상서비스>
  ./compose.sh up -d <대상서비스>

5) 운영 시 자주 발생하는 실수
- release-state.env 백업 없이 태그를 덮어씀
- 변경되지 않은 서비스 태그까지 불필요하게 수정함
- pull 없이 up만 실행해서 이전 캐시 이미지 실행
- latest 태그를 다시 사용함
- 헬스체크 확인 없이 배포 완료로 판단함
